# Contributing to Project Name

First off, thank you for considering contributing to this project! It’s people like you who make the open-source community such an amazing place to learn, inspire, and create.

## 📋 Code of Conduct
By participating in this project, you agree to abide by our project's standards: being respectful, using inclusive language, and accepting constructive criticism.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Check the **Issues** tab to see if the bug has already been reported.
- If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce the bug.

### Suggesting Enhancements
- Open an issue to discuss the idea before starting the work.
- Explain the "why" and "how" of the proposed feature.

### Pull Requests (PRs)
1. **Fork** the repository and create your branch from `main`.
2. Install dependencies using `make install`.
3. Make your changes, ensuring you follow the project's `.editorconfig` rules.
4. **Test your changes** by running `make test`.
5. Ensure your code passes the linter with `make lint`.
6. Submit a Pull Request with a clear description of your changes.

## 🛠️ Development Workflow
We use a **Makefile** to standardize the environment. Please use these commands:
- `make install`: Set up your local environment.
- `make lint`: Check code style and quality.
- `make test`: Run the test suite.
